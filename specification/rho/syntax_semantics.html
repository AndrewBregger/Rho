Variables

Virtually all programs that are not 'Hello World' uses variable. A variable is a construct within a program that binds a value to a name, and that name can be manipulated later within the program. A variable can be declared outside of a function or struct, just like in C/C++.

Declaring a varable x of type integer of 32 bytes.
x := 5;

The variables type is inferred from what it is begin assiged with. So in this case, the type is inferred to by an integer and the default integer type is i32. Which is an integer of 32 bytes.

Type annotations

A variable can be told what type it is when it is declared, this is not neccessary but it does make the code more readable.

Declaring a varable x that will be a floating point value of 32 bytes.
x: f32 = 5;

Because the variables type is annotated then the type will be casted to that type if it can. If it cannot be casted, a type error will be thrown.

Constant Variables

Most likly will change:
Variables that are not going to change throughout the run time of the program are considered, constant.

Therefore, to declare a constant:
x :: 5; // with type inferencing
x : f32 :: 5; // without type inferencing

Once this variable is declared, it can not be changed. If you mistakenly try to change it there will be an semantic error.

Functions

Every Rho program must have a main function:

main :: () {
}

This is the simplest function. It doesn't have any parameters and it doesn't return any values. The body of the function is between the curly brackets. Here's a function named foo:

foo :: () {
}

So, how are arguments handled? Here is a function that prints a number:

print_number :: (x: i32) {
printf("x is %1");
}

Return Values:


add :: (x: f32, y: f32) -> f32 {
return x + y;
}

Multiple Return Values:

foo :: (x: f32, y: f32) -> f32, f32 {
return x*2, y*2;
}

Associated Return Values:

foo :: (x: f32, y: f32) -> a: f32, b: f32 {
a = x*2;
b = y*2;
}

This method is associating the values of the return variables to a variable. These variables are able to be assigned to but not amipulated. If this is used, then a return statement is not used. However, all return variables must be assigned.

Looping

loop

Infinate loop.
loop {
if x == 4
break;
}

for

C-like for loop and for each loop.
b : [f32] = {1.0, 2.0, 3.0, 4.0};
for i := 0; i < b.size; ++i {
x := b[i];
}
for i : [0..10] {
x := b[i];
}
for i : b.iter() { // or maybe just b
x := i;
}

while

c-like while loop.
while y != 4 {
y++;
}
